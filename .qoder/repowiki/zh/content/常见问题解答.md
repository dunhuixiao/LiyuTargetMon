## 图标不显示

### 可能原因
1. **未正确启用监控条目**：导入配置后，部分监控条目可能未被激活。
2. **文件编码问题**：手动编辑 `.jx3dat` 文件时，若保存为 UTF-8 编码而非 GBK，会导致乱码和图标无法显示。
3. **缓存未刷新**：游戏或插件缓存未及时更新，导致旧配置仍被使用。

### 解决方案
1. 在「茗伊插件集 - 目标 - 目标监控 - 打开设置面板」中，找到任意一行监控条目，取消勾选“启用”后再重新勾选，强制刷新显示。
2. 确保所有手动编辑的 `.jx3dat` 文件以 **GBK 编码**保存。使用 VS Code 编辑时，点击右下角编码提示，选择“通过编码重新打开” → “Chinese Simplified (GBK)”。
3. 尝试重启游戏或使用“检查更新”功能重新加载最新配置。

## 自定义监控条目失败

### 可能原因
1. **操作路径错误**：未在正确的界面进行添加操作。
2. **权限或通用数据设置冲突**：启用了“使用通用数据”后，个别角色的自定义设置可能被覆盖。

### 解决方案
1. 添加自定义监控条目的正确方式：
   - 方法一：进入「茗伊插件集 - 目标 - 目标监控 - 打开设置面板」，选择目标监控集合，点击「新建条目」创建。
   - 方法二：在监控条上找到想要插入的位置，右键选择“插入”。
2. 若已启用“使用通用数据”，请先取消勾选该选项，完成自定义后再重新启用，以确保配置同步。

## 显示顺序错乱

### 可能原因
1. **手动编辑格式错误**：直接修改 `.jx3dat` 文件时，Lua 数据结构格式错误或花括号范围不正确。
2. **未按预期拖拽排序**：游戏内拖拽操作未生效或位置不准确。

### 解决方案
1. **推荐方法：游戏内拖拽调整**
   - 进入「茗伊插件集 - 目标 - 目标监控 - 打开设置面板」，直接拖动监控条目至目标位置。
   - 或在目标位置右键选择“插入”新条目。
2. **高级方法：编辑器修改原始文件**
   - 使用支持 GBK 编码的编辑器（如 VS Code、Notepad++）打开 `.jx3dat` 文件。
   - 按 Lua 格式调整 Buff 顺序，注意保持花括号 `{}` 的正确嵌套。
   - 修改完成后，务必以 **GBK 编码**保存，避免乱码。
   - 可参考 [GitHub 原始文件](https://github.com/dunhuixiao/LiyuTargetMon/tree/master/docs/.vuepress/public/targetmon/prioritylevel) 中的分类与注释进行调整。

## 界面风格异常

### 可能原因
1. **右对齐导致图标溢出**：当监控条设置为右对齐时，因总长度约为屏幕宽度的 80%，起始位置约在右侧 40%，可能导致部分图标超出屏幕范围而不可见。
2. **UI 缩放设置不当**：图标或文字大小缩放比例设置过高或过低，影响显示效果。
3. **主题或插件版本不兼容**：使用了不兼容的茗伊插件版本或自定义主题。

### 解决方案
1. 若使用右对齐导致图标不显示，建议将监控条位置调整至屏幕**中间或左侧**，避免溢出。
2. 调整显示参数：
   - 进入「茗伊插件集 - 目标 - 目标监控 - 打开设置面板 - 设置」。
   - 修改“图标缩放”、“图标文字缩放”、“其他文字缩放”以适配个人视觉需求。
3. 确保使用最新版本的茗伊插件集，并通过游戏内订阅或官方渠道下载最新版“鲤鱼监控”配置文件。

## 进阶使用提示

### 自定义监控建议
- **添加专属监控**：可通过查找技能 Buff ID，在设置面板中手动添加个性化监控条目，适用于特定流派或战术需求。
- **分类管理**：建议按 Buff 类型（如增益、减益、控制、免控）进行分组，便于快速识别。
- **别名优化**：为复杂技能设置简洁明了的提示名，提升战斗中信息获取效率。

### 版本与兼容性
- 本监控配置基于「茗伊插件集 - 目标监控模块」开发，遵循 MIT 开源协议，支持分享、修改与二次创作。
- 项目依赖 Node.js >=20.6.0 环境进行文档构建与维护，确保开发与发布一致性。
